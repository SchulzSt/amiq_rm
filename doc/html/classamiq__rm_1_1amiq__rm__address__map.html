Detailed Description

This class is used to model an address_map. An address map may contain registers and sub-maps, both categories are mapped by offset. It provides mechanisms to perform recursive search of the registers (by name, by offset).

void amiq_rm::amiq_rm_address_map::build ( )
virtual

The function must be called after all registers and all sub-maps have been added. It will recursively descend and call build() for mapped registers and sub-maps, connect parent pointers, etc. It is not necessary for the build() to be called if a parent map calls it's build().

amiq_rm_reg * amiq_rm::amiq_rm_address_map::get_reg_by_offset ( amiq_rm_reg_address_t  my_offset)

The function returns a pointer to the register specified by offset. The search will take place within directly mapped registers, but will also continue recursively through sub-maps. If the instance of the address_map in which this function is called is a top level map (associated with a physical interface), the offset will basically be the absolute address.

void amiq_rm::amiq_rm_address_map::reset ( )

The function implements the reset functionality for the address map. When this function is called, it will trigger reset() for the registers mapped in regs, also it will trigger reset() in each sub-map mapped in sub-maps.

amiq_rm_reg_map_t amiq_rm::amiq_rm_address_map::regs

The map contains pointers to the registers added to the map by using the offset as the key. Functions like get_reg_by_name() relies on this map. To add a register the user must use add_reg().

amiq_rm_addressmap_map_t amiq_rm::amiq_rm_address_map::submaps

The map contains pointers to the sub-maps added by using the offset as the key. Functions which iterate recursively like get_reg_by_name() relies on this map. To add a sub-map the user must use add_map().
